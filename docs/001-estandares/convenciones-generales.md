# Convenciones generales de SIGOM-ENOSA

## Idioma

- El código se escribe en inglés.
- Los nombres de variables, clases, métodos, rutas y tablas se escriben en inglés.
- La documentación funcional puede escribirse en español.
- Los mensajes visibles al usuario se escriben en español.

## Nombres de carpetas y archivos

- Las carpetas usan `kebab-case`.
- Los archivos usan `kebab-case`.
- No se usan espacios, tildes, ñ ni caracteres especiales.
- Los nombres deben expresar su propósito.

## Ejemplos válidos

- `work-orders`
- `create-work-order.dto.ts`
- `work-order-status.enum.ts`
- `inspection-evidence.service.ts`

## Ejemplos no válidos

- `Ordenes Trabajo`
- `ordenTrabajo.ts`
- `nuevo_archivo.ts`
- `OrdenTrabajoFinal.ts`

## Idioma del dominio

Términos técnicos principales:

- Work Order = Orden de trabajo
- Inspection = Inspección
- Evidence = Evidencia
- Technician = Técnico
- Crew = Cuadrilla
- Preventive Maintenance = Mantenimiento preventivo
- Supply = Suministro
- Meter = Medidor
- Incident = Incidencia
